我们的核心精神很简单：
- **公平启动**: 无预挖，无团队份额，无风险投资。从第一天起，每个人都有相同的机会进行铸币。项目团队唯一的优势，就像其他任何用户一样，仅仅是早期参与。

- **战略深度**: 我们设计了多层博弈论机制，包括一个新颖的惩罚系统和动态奖励，旨在为参与者创造一个更丰富、更具吸引力的体验。

### 3.2. 与 XEN 一致的“时间公平”排名模型
我们的核心奖励计算模型，是有意地与经过市场验证的、原始 XEN Crypto 的经济原则保持一致。该模型强调**“时间公平性”**，旨在创建一个系统，在该系统中，用户长期的承诺（耐心）是比其单纯的运气或速度（获得一个早期排名）更强大的、决定奖励的因素。

这一设计哲学确保了本协议对新来者具有永久的吸引力，因为它为他们提供了一个清晰的、可实现的策略，来与最早的采用者竞争，甚至超越他们。在我们协议中，最核心的战略杠杆，不是你*何时*加入，而是你*愿意承诺多长时间*。

- **公式核心**: 最终奖励由一个可被简化的公式驱动：
  `奖励 ≈ log2(全局排名 - 你的排名) * 期限 * 其他因子`

- **关键组成部分**:
    - **`cRank` (你的排名)**: 你在发起铸币那一刻所获得的全局排名编号。
    - **`globalRank` (全局排名)**: 协议中已被领取的总排名数量，它持续增长，代表着网络的整体成长和采用情况。
    - **`term` (期限)**: 你选择锁定你铸币仓位的天数。**这是你作为用户，所能控制的最强大的变量。**
    - **`log2` (以 2 为底的对数)**: 这是模型的心脏。这个数学函数会极大地削弱 `全局排名` 和 `你的排名` 之间的原始*线性差距*。这一数学特性，既是模型公平性的关键，也是其强大的、抗通胀特性的核心。

- **经济学意义**:
    - **耐心高于排名**: 我们的模型证明，一个较晚的参与者（例如, `cRank`: 7,000）如果选择一个更长的 `期限`（例如, 75 天），他可以获得比一个先驱用户（`cRank`: 1，但选择了较短的 `期限`，例如 45 天）高得多的总收益。**耐心比排名更具价值。**
    - **奖励网络增长**: 该公式确保了所有参与者都能从协议的整体成功中受益。随着 `全局排名` 的增加，`log2(全局排名 - 你的排名)` 这一部分对于每个人都会增长，创造了一个“水涨船高”的效应。这使得个人激励与网络扩张的集体目标保持一致。
    - **战略性博弈**: 每个用户在选择他们的 `期限` 时，都是在对协议的未来增长进行一次战略性的押注。一个更长的 `期限` 是在押注网络将会显著增长，从而为他带来远超流动性成本的、巨大的“网络增长”红利。这创造了一个强大且可持续的经济反馈闭环。

## 4. “强力唤醒”惩罚模型 (10 天窗口期)

为了鼓励积极参与并防止奖励被锁在“死钱包”中，我们对在铸币期限成熟后延迟领取奖励的行为设置了惩罚。这个模型被设计成具有战略性。

- **第一阶段: 休息期 (成熟后 1-2 天)**
    - **惩罚**: 0%
    - **理由**: 为用户提供一个宽限期，让他们可以无压力地领取。

- **第二阶段: 战略窗口期 (成熟后 3-5 天)**
    - **惩罚**: 每日线性增加 **5%**。
        - 第 3 天: 5%
        - 第 4 天: 10%
        - 第 5 天: 15%
    - **理由**: 这是核心的博弈论窗口。如果一个用户预测 `全局排名` 的增长将带来超过 5% 每日惩罚的收益增长，那么延迟领取就成了一个可行的策略。

- **第三阶段: 唤醒期 (成熟后 6-10 天)**
    - **惩罚**: 每日线性增加 **15%**，变得越来越严厉。
        - 第 6 天: 30%
        - 第 7 天: 45%
        - 第 8 天: 60%
        - 第 9 天: 75%
        - 第 10 天: 99%
    - **理由**: 战略窗口正在关闭。此时的惩罚已远远超过任何潜在的 `全局排名` 增长带来的收益，强烈地暗示用户必须立即领取。

- **10 天后**:
    - **惩罚**: 永久锁定在 **99%**。奖励实际上已被没收（6%直接分配给奖励池，剩余销毁），并且NFT变为可清算状态，任何用户都可以直接清算获得剩余的1%代币。

### 4.1. 惩罚的再分配
从被惩罚的奖励中获得的一部分价值不会被销毁，而是被再分配给活跃的质押者，创造了一个循环的经济闭环。

- **机制**: 当一个用户领取一笔会产生惩罚的奖励时，被罚没的（未被铸造的）$SLEEPING 代币总量的 **6%**，将被直接铸造到 `StakingRewards`（质押奖励）合约中。这部分金额随后将按比例分配给所有活跃的质押者，为协议提供了第二条以原生代币计价的奖励流。
- **例子**: 如果一笔 1,000,000 $SLEEPING 的奖励产生了 90% 的惩罚，那么 900,000 $SLEEPING 将被罚没。其中，6%（即 54,000 $SLEEPING）将被发送到质押合约进行再分配。剩下的 846,000 $SLEEPING 将永远不会被铸造，扮演了一个强大的通缩角色。

## 5. 费用分配与动态协议引擎

为确保协议的长期可持续性并创造一个强大的、具备自适应能力的经济引擎，所有协议产生的费用（来自铸币的 OKB，来自交易税的 $SLEEPING）都将由 `TreasuryDistributor`（财库分配器）合约管理。该合约采用一个基于协议增长率的动态分配模型，使其能够自动调整策略，在“巩固市场稳定”和“最大化参与者回报”之间取得平衡。

### 5.1. 16 天周期 (Epoch)
协议的运行被划分为连续的、每期 16 天的周期（Epoch）。在每个周期结束时，合约会衡量该周期内 `全局排名` 的总增长量。这个增长数据将决定下一个完整的 16 天周期内，费用分配的固定比例。这种周期性的调整，使得协议能够以一种稳定且可预测的方式，对市场趋势做出反应。

### 5.2. 动态分配模式
费用的分配策略由上一个周期的 `全局排名` 增长量决定，并在两种主要模式之间进行线性插值计算：

- **“寒冬模式” (低增长)**: 当 16 天周期内 `全局排名` 增长量**低于 5,000** 时触发。这是一种旨在市场冷静期巩固协议基本面的防御性策略。
    - **55% -> 协议拥有的流动性 (POL)**: 大部分收入被用于加深流动性池，以巩固代币的价格底部。
    - **40% -> 质押奖励**: 确保为忠诚的质押者提供持续且有吸引力的收益。
    - **5% -> 回购与销毁**: 维持基础的通缩压力。

- **“牛市模式” (高增长)**: 当 16 天周期内 `全局排名` 增长量**高于 50,000** 时触发。这是一种旨在高活跃度时期最大化社区回报和通缩效应的进攻性策略。
    - **10% -> 协议拥有的流动性 (POL)**: 持续构建流动性基础，确保在快速增长时期的市场稳定性。
    - **70% -> 质押奖励**: 绝大部分收入被重定向给社区，以奖励其积极参与。
    - **20% -> 回购与销毁**: 利用高额的协议收入，加速代币销毁，放大价值累积效应。

- **“常规模式” (正常增长)**: 当 16 天周期内 `全局排名` 增长量介于 **5,000 和 50,000 之间**时，POL、质押奖励、回购销毁的分配比例，将通过线性插值公式，在“寒冬模式”和“牛市模式”的设定值之间进行平滑计算。这确保了协议经济策略的连续、平稳调整。

## 6. 质押奖励 & 分红系统 (The Share System)

质押 $SLEEPING 允许用户通过一个复杂的、基于分红权（Shares）的系统来赚取协议的 OKB 收入。该系统旨在奖励长期的承诺。当您质押您的 $SLEEPING 时，您会获得相应的**分红权 (Shares)**。您所拥有的分红权相对于总量的比例，决定了您在 OKB 奖励池中所占的份额。

### 6.1. 滚仓分红周期
协议的 OKB 收入（即分配给质押奖励的部分）不会每周进行一次性派发，而是被持续地注入到多个不同周期的滚仓奖池中。这些奖池会在指定的日期自动向所有分红权持有人支付奖励，永久循环。

滚仓分红周期运行如下：
- **每 6 天** (例如, 第 6, 12, 18... 天)
- **每 30 天** (例如, 第 30, 60, 90... 天)
- **每 90 天** (例如, 第 90, 180, 270... 天)
- **每 360 天** (例如, 第 360, 720, 1080... 天)
- **每 720 天** (例如, 第 720, 1440, 2160... 天)

交易税及铸造费的OKB收入将按以下比例分配到这些周期奖池中：
- **25%** 进入 6 天滚仓奖池。
- **25%** 进入 30 天滚仓奖池。
- **17%** 进入 90 天滚仓奖池。
- **17%** 进入 360 天滚仓奖池。
- **16%** 进入 720 天滚仓奖池。

这个系统创造了持续的市场预期，并战略性地让多个周期的奖励在同一天触发，形成"**超级分红日**"：

- **第30天**: 6天池 + 30天池 同时分红
- **第90天**: 6天池 + 30天池 + 90天池 同时分红
- **第360天**: 6天池 + 30天池 + 90天池 + 360天池 同时分红
- **第720天**: 🎉 **超级大满贯** - 全部5个池子（6+30+90+360+720天）同时分红！

这些同步的分红事件创造了巨额奖励日，激励长期承诺。

### 6.2. 如何赚取分红加成
您获得的分红加成数量取决于您开始质押时的**分红权汇率 (Share Rate)**。为了激励早期参与者，分红权汇率在协议启动的第一年内是动态变化的：
- **早期激励**: 协议上线第一天，汇率为 **1.1x** (即 1 SLEEP = 1.1 分红权)。
- **线性过渡**: 这个 10% 的额外奖励将在 365 天内平滑地、线性地衰减。

**BiggerBenefit计算规则**：
- 每个Access Pass NFT最多记录6个独立存款
- 卡内BiggerBenefit = MAX(单次存款最高等级, 卡内总额等级)
- 同一张卡内的所有存款都享受该卡的最高BiggerBenefit等级

**完整APY计算公式**：
```
总APY = 基础APY(100%) + 实际天数奖励(最高206%) + BiggerBenefit(最高6%) + 无限模式奖励(366%，如适用)

普通模式最高：100% + 206% + 6% = 312%
无限模式最高：100% + 366% + 6% = 472%
```

质押权重：用户并非所有时长进入质押都会收到相应池子的奖励，每比质押只会在允许时间内作为一个批次， 进入窗口是奖励的3/1时间内，如6日分红，用户在前两日进入，则可以享受六日分红，如果不满足3/1机制，则会进入下一轮。该用户虽然不满足6日质押池，但是30,90,360,720都满足，也就是说 ，如果用户想要吃720大奖，就要在240天内加入，用户也不至于纠结，因为即使超出了240天，依旧还有其他池子可以分享。

### 6.3. 质押惩罚机制
该系统旨在对长期参与者保持公平。
- **最低质押期**: 26 天。
- **提前结束质押**: 在质押期未满 50% 时，不允许结束质押。在质押期超过 50% 但未到期时结束，您质押的本金将被**惩罚性罚没 55%，其中49%直接销毁，6%奖池作为奖励用户资金**，您将收回剩余 45% 的本金。
- **延迟结束质押**: 质押到期后，您有 **6 天的宽限期**可以无惩罚地结束质押。超过宽限期后，您的本金将每天被罚没 **1.45%**。当累计罚金超过本金的 **50%** 后，该质押仓位将变为可被任何其他用户"清算"的状态。清算时产生的总罚金，其中 **6%** 将被发送到质押奖励池，**2%** 将给予清算人，余下的 **92%** 将被销毁。原质押者将收回其剩余本金。最大罚没上限为 **96%**。

### 6.4. 多次质押的取出机制

- 每个存款记录独立计算罚没时间

### 9.6. 完整奖励计算示例

#### **铸币奖励计算公式**
```solidity
// 最终奖励 = log2(globalRank - userRank) × 快照amplifier × term × eaaRate × 时间衰减因子

function calculateMintReward(
    uint256 userRank,
    uint256 globalRank, 
    uint256 term,
    uint256 count,
    uint256 amplifier,
    int128 timeDecayFactor
) returns (uint256) {
    // 1. 计算Log2差值
    uint256 rankDiff = globalRank - userRank;
    uint256 log2RankDiff = log2(rankDiff);
    
    // 2. 计算EAA (Early Adopter Advantage)
    uint256 eaaRate = count + 1;
    
    // 3. 组合计算
    uint256 baseReward = log2RankDiff * amplifier * term * eaaRate;
    
    // 4. 应用时间衰减
    int128 finalReward = ABDKMath64x64.fromUInt(baseReward).mul(timeDecayFactor);
    
    return ABDKMath64x64.toUInt(finalReward);
}
```

#### **质押APY计算公式**
```solidity
// 总APY = 基础APY + LongerPaysMore奖励 + BiggerBenefit奖励 + 无限模式奖励

function calculateStakingAPY(
    uint256 stakingDays,
    uint256 amount,
    bool isInfinite
) returns (uint256) {
    uint256 totalAPY = 100; // 基础100% APY

    if (isInfinite) {
        totalAPY += 366; // 无限模式366%
    } else {

    return totalAPY;
```
