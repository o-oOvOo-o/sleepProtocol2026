# Sleep Protocol - 任务清单

本文件用于跟踪 Sleep Protocol 项目剩余的开发与错误修复任务。

## 🟥 优先级 1: AMM 池重构与实现

-   [ ] **重构 `SleepV2Pool.sol` 至 Uniswap 标准**:
    -   **任务**: 修改 `SleepV2Pool.sol` 和 `SleepRouter.sol` 的交互模式，使其完全遵循 Uniswap V2 的标准实现（由 Pool 从 Router `transferFrom` 代币，而不是 Router 主动 `transfer`）。
    -   **理由**: 统一代码标准，消除潜在的、由非标准交互模式带来的未知风险，并与行业最佳实践保持一致。
-   [ ] **完整实现 `SleepV4Pool.sol`**:
    -   **任务**: 移除所有简化的、用于演示的占位符逻辑。
    -   **任务**: 基于你提供的 Uniswap V4 核心代码，为其实现完整的集中流动性 AMM 功能，包括但不限于：
        -   精确的 `TickMath` 库集成。
        -   可靠的流动性头寸管理 (`mint`/`burn`)。
        -   高效、准确的核心交换算法 (`_executeSwap`)。
    -   **目标**: 使其成为一个功能完备、安全可靠、媲美 Uniswap V4 的 SLEEP/OKB 专用集中流动性池。

## 🟧 优先级 2: 关键 Bug 修复

-   [ ] **修复质押合约的 `SLEEP` 奖励分配**: `TokenStaking.sol` 合约目前缺少一个机制，来将从罚金中收到的 `SLEEP` 代币分配到各个分红池中。这是一个与白皮书不符的关键逻辑缺陷。
    -   **任务**: 在 `TokenStaking.sol` 中实现一个新函数 (例如 `depositSleepReward`)。
    -   **任务**: 确保该函数能按照白皮书的规定，将 `SLEEP` 代币分配到 30天、90天、360天和720天的奖池中。
    -   **任务**: 更新 `TokenMinter.sol` (或其他产生罚金的合约)，使其调用这个新函数，而不是执行简单的 `transfer`。

## 🟨 优先级 3: UI/UX 重构

-   [ ] **以 NFT 为中心重新设计 `Stake.tsx` 页面**: 当前的质押页面 UI 与基于 Access Pass NFT 的质押模型不匹配。
-   [ ] **在 UI 中展示详细的奖池奖励**: UI 应该能同时展示 `OKB` 和 `SLEEP` 的奖励余额。

## 🟦 优先级 4: 代码审查与验证

-   [x] **初步审查 AMM 池合约**: 已完成初步审计，发现了非标准交互模式和 V4 逻辑缺失的问题。*(重构和实现后需要进行最终审查)*。
-   [ ] **审计用户清算逻辑**: 审查 `TokenStaking.sol` 和 `TokenMinter.sol` 中处理用户清算的部分。
-   [ ] **验证前端与池的集成**: 在 AMM 池重构完成后，确保前端能正确地与 Swap 路由器和池进行交互。

## 🟩 优先级 5: 部署与最终验证

-   [ ] **最终部署**: 在所有修复和重构完成后，执行一次完整的、干净的部署到 `xlayertest` 网络。
-   [ ] **最终验证**: 在部署后，对所有功能进行彻底测试。
